Changelog

Items listed in bold are considered potentially breaking changes from the previous released version. Potentially breaking changes include:

  • Changes to the structure of returned objects
    • Addition/removal of properties or links
    • Changes to the structure of composed types (e.g. collections)
  • Status code changes
  • Changes to default parameter values

We make a special effort not to remove properties or links, change the structure of composed types, change the class of status codes (e.g. 200 class to 400 class), or change default parameter values.

Clients should be prepared to handle changes to status codes within the same class, especially the 400 and 500 classes, and to handle the addition of properties and links to response objects.

PI Web API Indexed Search 2018 SP1

New features:

  • None

Fixes and Enhancements:

  • Endpoints used by the crawler are secured
  • Get Children searches can optionally return unsorted results, improving query performance
  • Renamed search sources are no longer added as duplicates
  • Query results are not corrupted when a database is renamed to an already indexed database between crawls

Data model changes:

  • None

Other behavior changes:

  • None

PI Web API Indexed Search 2018

New features:

  • None

Fixes and Enhancements:

  • Search sources with Unicode names operate normally after PI Web API service restarts
  • PI Point attributes changedate and creationdate are indexable and searchable
  • Missing or malformed Sources.txt file does not prevent PI Web API from starting

Data model changes:

  • None

Other behavior changes:

  • Indexed search crawler supports TLS 1.2 by default

PI Web API Indexed Search 2017 R2

New features:

  • Full support of FIPS
  • Support WebID 2.0 (type "Full" only)

Fixes and Enhancements:

  • Optimized document load time when specific fields are selected in search query
  • Incremental crawl now picks up changes for elements based off of derived templates
  • Elements with only ReadData permission are now searchable
  • Fixed matchedfields calculation for elements with a large number of attributes

Data model changes:

  • None

Other behavior changes:

  • None

PI Web API Indexed Search 2017

New features:

  • Provides progress status during the finalize portion of crawl operations

Fixes and Enhancements:

  • Added numeric crawl indicator on the Admin page
  • Fixed path calculation during incremental crawl when a referenced element is added as a composition reference type
  • Fixed handling of malformed queries

Data model changes:

  • None

Other behavior changes:

  • None

PI Web API Indexed Search 2016 R2

New features:

  • FIPS support CTP - PI Web API Indexed Search will automatically switch to FIPS compliant algorithms if Windows is configured to use them.

Fixes and Enhancements:

  • Improved scheduling of simultaneous crawls now permits concurrent crawls up to the configured limit of MaxConcurrentCrawlers.
  • WebId added to search source.
  • Improved incremental crawl times for PI Data Archive.
  • General bug fixes, security enhancements, appearance, and performance improvements.

Data model changes:

  • "WebID" is now spelled "WebId" in search results for consistency with the rest of PI Web API.

Other behavior changes:

  • None

PI Web API Indexed Search 2016

New features:

  • Shared Index CTP functionality. See "PI Web API Index Search 2016 - Shared Index CTP" white paper for more details.

Fixes and Enhancements:

  • Renaming a parent no longer clears the references to child elements.
  • Specifying an unconfigured search source in a query now returns an ElementNotFound error instead of an empty result.
  • AF Server memory is no longer exhausted when a large number of Event Frames are present.
  • Resolved an issue where Coresight reported "The Server returned a bread crumb with an undefined path" when adding a search source.
  • Resolved an issue where the PI Web API Crawler service may miss updates due to read permissions.
  • Queries for Event Frames can be made by calling PI Web API. Earlier undocumented methods exposed by search API will now return warnings. These methods will be removed entirely in a future release.
  • Index finalization is now canceled on rebuild requests.
  • Support for PI AF 2.8.
  • General bug fixes, security enhancements, appearance, and performance improvements.

Data model changes:

  • None

Other behavior changes:

  • PI Data Archive connections are now closed when their search sources are removed.
  • Child query results are now sorted by name.
  • An event log entry will be generated if you are using a NETBIOS name instead of the fully qualified domain name. Using a NETBIOS name instead of a Fully Qiualified Domain Name can create communication issues between the PI Web API Service and the PI Web API Crawler Service.

PI Web API Indexed Search 2015 R3

New features:

  • Search for Attributes by PI Point Data Reference
  • Improved performance while calculating paths for Asset databases when using AF Server 2.6 or later.
  • Search by static attribute value.
  • The ability to retrieve WebIds for Event Frames.
  • Retrieve First and Last page results directly.
  • Search by attribute categories.
  • Crawl progress bar on Admin page.
  • Improved logging.

Fixes and Enhancements:

  • Corrected RelatedEvents paging for overlapping Event Frames.
  • Updated help documentation.
  • Querying for wildcard (*) no longer throws a matched field error.
  • Fixed an issue with matched fields when using the + operator.
  • Corrected the behavior that ignored new PI Points on incremental crawls.
  • General bug fixes, security enhancements, appearance, and performance improvements.

Data model changes:

  • None

Other behavior changes:

  • None
